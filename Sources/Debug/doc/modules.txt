    чтения сивола, перевода кода клавиши в строку и обратно.

    Перед использованием необходимо вызвать процедуру Ini, по окончании Exi;

    - Обработсиу диалогового окна добавления останова по доступу

Пакетный отладчик:

  pack:
    Содержит процедуру для запуска пакетного отладчика.
    Имя исполняемого пакета должно быть определено в модуле
    Options, переменная tst_name.

  pcktypes:
    Модуль содержит описание типов и переменных, используемых пакетным
    отладчиком, в частности, определена структура для пакетов, 
    режимы работы паетного отладчика, описание точек останова
    и идентификаторов.

  pckbase:
    Содержит набор процедур, используемых пакетным отладчик для 
    интерпретации апкета, задания и проверки режимов работы,
    определения идентификаторов для режимов и способов их переключения.

  pcklists:
    Модуль для организации работы (добавление, проверка, удаление)
    с метками, со стеком вызовов процедур пакета, использования эквивалентных
    имен, обработке ошибок в пакете.

  pckopers:
    Интерпретация команд языка пакетной отладки.

  pckreact:
    Содержит процедуры, реализующие реакции на происходящие события
    в программе: исключительные ситуации, точки останова, установку
    и снятие остановов пакетного отладчика.
 
  pckmodel:
    Содержит процедуры для отображения результатов моделирования
    ввода/вывода, процедур, доступа к памяти.

  errors:
    Выдача ошибок при работе в пакетном отладчике.

Конечный исполнитель (связь с интерепретатором)

  krntypes:
    Определение основных типов данных целевой машины
    необходимых для работы системы отладки.

  krnexec:
    Загрузка программы, отгрузка, рестарт, остановы, исполнение,
    режимы исполнения, получение времен исполнения. Модуль
    является системо-независимым.
   
  krn_prog:
    Загрузка программы, отгрузка, рестарт, остановы, исполнение,
    режимы исполнения, получение времен исполнения. Модуль
    является системо-зависимым, использует функции интерпретатора.

  krn_mem: 
    Процедуры для доступа к регистрам и памяти (чтение/запись/атрибуты).
    Также содержит функции для установки/снятия трассировки
    регистрам и областей памяти.

  krn_dasm: 
    Предоставляет процедуры для диассемблирования команды
    по заданому адресу.

  krnindex:
    Организация работы с остановами. Содержит процедуры для добавления,
    поиска и удаления остановов.


Работа с отладочной информацией:

  di_types:
    Описание всех используемых типов во внутреннем универсальном
    представлении отладочной информации: компонент, модулей,
    типов, обьектов.

  di_build:
    Предоставляет процедуры для преобразования и постороения
    внутреннего универсального представления отладочной информации.

  di_tools:
    Функции для доступа к универсальному внутреннему представлению.
    Содержит процедуры для поиска разнообразных обьектов, получения
    их атрибутов. Работа с компонентами, модулями, процедурами,
    переменными (глобальными и локальными), типами, обьектами,
    получение исходных текстов модулей, поиск соответствия между
    разнообразными структурами (строками и кодом, адресами и обьетами).

  di_read:
   Чтение отладочной информации для программы, преобразование
   внешнего представления отладочной информации во внутреннее.


Вычислитель выражений:

  expr
    Содержит процедуры, с помощью которых пользователю предоставляется
    возможность при отладке программы работать с выражениями языка
    программирования в привычном для него виде. В качестве средства
    разработчика, предоставляет набор функций, предназначенных для разбора
    строкового представления выражения и вычисления его результата, получения
    разнообразных атрибутов логических объектов программы, доступа к внутренней
    структуре логических объектов, преобразования внутреннего представления
    объектов и результатов вычисления выражений в строковое представление.


Вспомогательные модули:

  dll:
    Поддержка использования библиотек динамического связывания.
 
  file: 
    Модуль для работы с файлами: преобразование имен и времен.
    
  msgno: 
    Определение номеров сообщений отладчика.
    
  options: 
    Набор переменных, определяющих режим работы отладчика.

  protocol: 
    Модуль управления созданием протокола отладки и выдачей сообщений.
    Cписок сообщений создается на основе указано текста.
    Поиск файла осуществляеться по red-файлу системы, если не найден -
    поиск продолжаеться в "домашней", т.е. откуда запущен.

  redfile: 
    Модуль, предоставляющий функции для работы с перенаправляющим
    файлом системы. Структура списков путей поиска создается на
    основе указанного файла.

  sort: 
    Процедуры для сортировки и поиска информации.

  texts: 
    Модуль предоставляет абстрактное понятие
    текста и набор процедур для работы с ним

  real2str:
    Преобразование длинных вещественных чисел в строковое представление.

  xcpt_msg:
    Функции для перевода кодов ошибок, возникающих при работе
    программы, в строковое представление (сообщение).

  help:
    Модуль предназначен для создания и выдачи помощи пользователю.

  xstr:
    Различные процедуры для работы со строками.

  xd_parse:
    Разбор параметров командной строки.

  convert:
    Преобразование чисел в строку и наоборот по заданной базе.
